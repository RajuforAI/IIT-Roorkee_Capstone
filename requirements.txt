# ---- Optional: Gemini fallback provider -------------------------------
# Migrated from the EOL'd `google-generativeai` SDK to `google-genai`
# per Issue #8 (2026-06-25); the legacy SDK reached end-of-life and

# ---- AWS SDK (Issue #12 — S3 upload path) -----------------------------
# Pinned to the README §3 version (1.35.45).  The S3 upload helper
# in ``telecom_rag.storage.s3`` is a lazy import — the rest of the

# that path must work from a fresh container with no extra step.
# See docs/issues/012-s3-upload-path.md.
boto3==1.35.45

# ---- /healthz endpoint (Issue #14 — observability smoke test) --------
# ``starlette`` powers the Starlette ASGI app that serves ``/_healthz``
# from ``telecom_rag.observability.health``. ``uvicorn`` is the ASGI
# server the production smoke job binds to (CI runs the same entrypoint:
# ``uvicorn telecom_rag.observability.health:app --port 8502``).
#
# Both are installed transitively today (``streamlit`` pulls starlette,
# ``chromadb + streamlit`` pull uvicorn) — but pinning them explicitly
# is the difference between a silent breaking-change when one of
# those deps drops the transit, and an install-time failure that
# the requirements.txt user catches immediately. The lazy-import guard
# in health.py (Starlette = None on ImportError) means a missing
# starlette surfaces ONLY on the first HTTP request to ``/_healthz``
# — exactly when an ALB is checking liveness. Pin it.
#
# Versions verified to install + import on Python 3.11 against the
# Issue #13 dependency set (langchain 1.3.x, langgraph 1.2.x).
# See docs/issues/014-healthz-in-ci.md.
starlette==1.3.1
uvicorn==0.49.0
